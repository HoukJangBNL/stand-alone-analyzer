### 2.2 미해결 root cause 2개

**Root #1 — `merged_m3.pt` weight format mismatch** (attempt 7 Job 12, 11s fail)
- `_resolve_merged_m3_path()`가 hit → `_run_sam_multi_gpu`이 `use_original_sam2=True` 분기 → vendor `build_sam2_from_yaml` → SAM2 native `_load_checkpoint`이 `sd["model"]` 강제 → KeyError.
- 우리 `merged_m3.pt` shape: `{model_config, model_state_dict}` (`build_merged_m3.sh` 산출).
- Vendor의 다른 경로 `build_sam2_finetuned` (LoRA-runtime, `run_amg_v2.py:578-582`)는 `model_state_dict`를 인식 — 즉 우리 shape 호환.
- **Fix 옵션**:
  - **A (권고)**: `_run_sam_multi_gpu`에서 `merged_m3` 경로 갈 때 `state_dict["model"] = state_dict.pop("model_state_dict")` reshape 한 줄 추가. vendor 코드 무변경.
  - B: `merged_m3` 라우팅 끄고 LoRA-runtime 강제 (vendor가 이미 우리 shape 인식). `use_original_sam2=False`로 환경 override. 단 LoRA-runtime은 §15에서 12.16 s/card-img (3.06× slower) — perf 손해.
  - C: `build_merged_m3.sh` 출력 shape를 `{"model": ...}`로 변경 → 재배포. 별건 작업 추가.

**Root #2 — cuDNN sublibrary version mismatch** (attempt 7 Job 13, 22.5s fail)
- AMI `ami-092ae5880cb9cf957`의 stack: driver 580.159.04 / CUDA 13.0 / `torch 2.12.0+cu130` / cuDNN 9.20.00.
- 첫 conv2d forward에서 `RuntimeError: CUDNN_BACKEND_TENSOR_DESCRIPTOR cudnnFinalize failed cudnn_status: CUDNN_STATUS_SUBLIBRARY_VERSION_MISMATCH`.
- §15 작동 시점은 다른 AMI (`ami-0b7ec5ff47a1eff11`, cu124 stable) — 그 AMI는 cuDNN ABI 정합. #228에서 DLAMI base 갈아끼우면서 깨짐.
- **Bake-time issue, runtime fix 불가**. AMI rebake 필요.
- **Fix 옵션**:
  - **A (권고)**: `requirements-inference.txt`에서 `torch` line을 stable cu124 또는 cu126로 pin → AMI rebake. 약 30-45분 + ~$0.50 spot.
  - B: DLAMI parent를 cuDNN-aligned 버전으로 교체 → AMI rebake.
