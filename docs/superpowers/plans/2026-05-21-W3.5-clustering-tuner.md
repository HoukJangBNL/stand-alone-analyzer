# W3.5 — Interactive Clustering Tuner UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Surface the W4.4 backend tunable `reg_covar` (default `10.0`) and the auto-optimiser `auto_tune_reg_covar` to the Clustering tab UI so the user can dial covariance regularisation with a log-scale slider and click "Auto-tune" to let the backend pick a value from the documented candidate set.

**Architecture:**
- **Backend (api-developer):** Add `reg_covar: float` to `ClusteringRefitParams`, default `10.0`, range `[0.0, 100.0]`. Add an opt-in `auto_tune: bool = False` flag on the same schema. The route forwards `reg_covar` to `run_clustering_step(...)` unchanged. When `auto_tune=True`, the route ignores the incoming `reg_covar` and instead loads the same RGB array the wrapper would build (via the existing engine helper), calls `auto_tune_reg_covar(points, seed_indices)`, fits with the chosen value, and emits the chosen value back to the client through the SSE `done` event payload (`result.reg_covar_chosen: float`).
- **Frontend (frontend-architect):** Add a "Covariance regularisation" log-scale slider to `<FitGMMButton>`'s host card (`ClusteringTab.tsx`), driven by a new Zustand field `regCovar` (default `10.0`). The slider uses log-space mapping for the range `[0.1, 10.0]` so the user can fine-tune at low values. Add an "Auto-tune" button next to it that submits `auto_tune=true`; on `done`, the slider snaps to `result.reg_covar_chosen` and a toast confirms the chosen value. Both controls are wired through `useClusteringRefit`.
- **No DB / no schema migration.** `reg_covar` is wrapper-level metadata recorded in `manifest.json::steps.clustering.params` (already enforced by W4.4). Nothing about Postgres `analyses` changes.

**Tech Stack:**
- FastAPI 0.110+, pydantic v2.6+ (`Field(ge=..., le=...)` for bounds).
- React 18.3, Zustand 4.5, Sonner 1.4 (toast on auto-tune completion).
- pytest 8.x + pytest-asyncio for the route test; Vitest 1.4 + @testing-library/react for the slider+button tests.
- httpx 0.28.1 `AsyncClient`+`ASGITransport` for the in-process SSE round-trip test.
- No new dependencies.

**Pre-read (do not modify):** `docs/superpowers/plans/2026-05-21-W4.4-clustering-spec.md` (decisions), `claudedocs/clustering-tunable-spec.md` (band rationale), `src/flake_analysis/core/clustering/auto_opt.py` (engine helper), `src/flake_analysis/api/sse.py` (`sse_stream`, `ProgressBridge`).

## Pinned decisions

These pin the design choices the user/PM has already made for W3.5. No later task may walk one back.

1. **Default `reg_covar=10.0` everywhere.** The pydantic schema, the Zustand slice, the slider initial position, and the slider's "reset" target all read the same constant `10.0`. This matches the W4.4 backend default (`engine.py`, `core/pipeline/clustering.py`, `pipeline/clustering.py`). No drift.
2. **Slider range `[0.1, 10.0]` (log scale).** The auto-opt candidate set in `auto_opt.py` is `{0.1, 0.3, 1.0, 3.0, 10.0}`. The slider must let the user reach every candidate. Below `0.1` the rank-deficient seed covariance leaks (W4.4 §"BLOCKED resolution"), above `10.0` the engine over-shrinks and unseeded blobs collapse — both empirically harmful, so the slider hard-clamps at the candidate band. The schema accepts `[0.0, 100.0]` so power users hitting the API directly are not blocked, but the UI does not expose values outside `[0.1, 10.0]`.
3. **Auto-tune is a SEPARATE button, not a slider toggle.** Dragging the slider always submits a manual fit with that exact value. Clicking "Auto-tune" bypasses the slider, runs the optimiser, and snaps the slider to the chosen value once `done` arrives. This keeps the manual control direct and the auto-tune path explicit.
4. **`auto_tune_reg_covar` runs server-side, NOT in the browser.** SciPy `cKDTree` + sklearn `GaussianMixture` belong on the backend. The client only sets `auto_tune=true` and consumes the `reg_covar_chosen` field on the `done` event.
5. **Auto-tune reuses the existing `/run/clustering/refit` SSE route — no new endpoint.** The `auto_tune` boolean is part of the existing request body. This keeps the SSE/lock/progress wiring intact and avoids a second route that duplicates 80% of the existing one.
6. **`reg_covar_chosen` is added to `ClusteringSummary`.** The schema gains an optional `reg_covar_chosen: float | None = None` field. When `auto_tune=False`, the route sets it to the incoming `reg_covar` (echoes back the value used). When `auto_tune=True`, the route sets it to the optimiser's pick. Either way the client always knows what value was actually used.
7. **`max_iter`, `tol`, `random_state` stay hidden.** They exist as engine parameters but are not exposed by the schema. We are not adding more knobs in W3.5.
8. **No backend mutation of `auto_tune_reg_covar`.** The optimiser already exists from W4.4. We import and call it; we do not refactor it. If a behaviour gap surfaces, it goes to W9 (real-data sweep), not W3.5.
9. **TDD on both sides.** Backend: pytest red→green for the schema validation, the SSE round-trip with `auto_tune`, and the `reg_covar_chosen` echo. Frontend: Vitest red→green for the slider rendering, `regCovar` slice updates, log-scale conversion, and the auto-tune button submitting `auto_tune: true`.

import numpy as np
import pandas as pd
from flake_analysis.core.clustering.auto_opt import auto_tune_reg_covar


def _build_auto_tune_inputs(manifest_obj, seed_groups_payload):
    """Mirror pipeline/clustering.py preconditions to call auto_tune_reg_covar."""
    af = Path(manifest_obj.analysis_folder)
    npz = np.load(af / "02_domain_stats" / "stats.npz", allow_pickle=False)
    rgb_all = npz["repr_rgbs"]
    flake_ids_all = npz["flake_ids"].astype(np.int64)
    sel = pd.read_parquet(af / "03_selector" / "selection.parquet")
    selected_ids = sel.loc[sel["selected"].astype(bool), "domain_id"].astype(int).to_numpy()
    id_to_pos = {int(d): pos for pos, d in enumerate(flake_ids_all)}
    keep = [id_to_pos[int(d)] for d in selected_ids if int(d) in id_to_pos]
    rgb_sel = rgb_all[np.array(keep, dtype=np.int64)]
    sel_ids = np.array([int(d) for d in selected_ids if int(d) in id_to_pos], dtype=np.int64)
    sel_id_to_pos = {int(d): pos for pos, d in enumerate(sel_ids)}
    seed_pos = []
    for grp in seed_groups_payload:
        pos = [sel_id_to_pos[int(d)] for d in grp["domain_ids"] if int(d) in sel_id_to_pos]
        if pos:
            seed_pos.append(pos)
    return rgb_sel, seed_pos


# Inside run_clustering_refit, replace call_wrapper:
chosen_reg_covar = params.reg_covar
if params.auto_tune:

// web/src/lib/logScale.ts
export function logToValue(t: number, min: number, max: number): number {
  const tt = Math.max(0, Math.min(1, t))
  return min * Math.pow(max / min, tt)
}

export function valueToLog(v: number, min: number, max: number): number {
  const vv = Math.max(min, Math.min(max, v))
  return Math.log(vv / min) / Math.log(max / min)
}

```tsx
// web/src/components/clustering/AutoTuneButton.tsx
import { useEffect, useRef } from 'react'
import { toast } from 'sonner'
import { useClusteringStore } from '@/state/clusteringSlice'
import { useClusteringRefit } from '@/hooks/useClusteringRefit'

interface Props {
  projectId: string
}

export function AutoTuneButton({ projectId }: Props) {
  const seedGroups = useClusteringStore((s) => s.seedGroups)
  const fitScope = useClusteringStore((s) => s.fitScope)
  const initialMaxMahalanobis = useClusteringStore((s) => s.initialMaxMahalanobis)
  const setRegCovar = useClusteringStore((s) => s.setRegCovar)
  const refit = useClusteringRefit(projectId)
  const lastSeenStatus = useRef<typeof refit.status>('idle')

  useEffect(() => {
    if (refit.status === 'done' && lastSeenStatus.current !== 'done') {
      const chosen = (refit.result as { reg_covar_chosen?: number } | null)?.reg_covar_chosen
      if (typeof chosen === 'number') {
        setRegCovar(chosen)
        toast.success(`Auto-tuned reg_covar = ${chosen.toFixed(2)}`)
      }
    }
    lastSeenStatus.current = refit.status
  }, [refit.status, refit.result, setRegCovar])

  const enoughGroups = seedGroups.length >= 2
  const busy = refit.status === 'running'
  const disabled = !enoughGroups || busy

  function handleClick() {
    refit.run({
      seed_groups: seedGroups.map((g) => ({ name: g.name, domain_ids: g.member_ids })),
      fit_scope: fitScope,
      max_mahalanobis: initialMaxMahalanobis,
      auto_tune: true,
    })
  }

  return (
    <button
      data-testid="clustering-auto-tune"
      type="button"
      onClick={handleClick}
      disabled={disabled}
      style={{ padding: '6px 12px' }}
    >
      Auto-tune{busy ? ` (${Math.round(refit.pct * 100)}%)` : ''}
    </button>
  )
}
```
