# W5-infra — S3 Uploads Bucket + CORS + IAM + Lifecycle (AWS provisioning plan)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **HARD RULE.** This plan creates billed AWS resources in account `931886963315` (`us-east-2`). The implementer subagent prepares JSON config files, the audit script, and the runbook. The implementer **does NOT run `aws ... put-*` / `iam create-*` commands**. Every state-change Step is marked with `🚨 USER APPROVAL REQUIRED 🚨` and is executed by the **PM** *only* after the user types "go" in response to a presented dry-run output.

**Goal:** Provision the AWS S3 bucket + CORS + IAM + lifecycle rules required by W5-B (presigned PUT) and W5-C (browser uploads). Single bucket `qpress-uploads` in `us-east-2`, two prefixes `dev/` and `prod/`, browser uses time-limited presigned URLs only (no AWS creds in the SPA), backend FastAPI authenticates with an EC2 instance role (or local IAM user during dev). All decisions are locked from the W5 D-block (2026-05-22) — see "Locked Decisions" below.

**Architecture:** Backend (FastAPI) holds AWS credentials (instance role on EC2, IAM user `qpress-dev-local` on the owner's Mac during dev). Browser receives short-lived presigned URLs from the backend and PUTs directly to S3 — bypassing the API for the bytes. SHA256 is computed in the browser and verified by S3 via `x-amz-checksum-sha256` (D2). Cross-prefix writes are denied at the bucket-policy layer using `aws:PrincipalTag/Env`, so even an over-broad IAM policy can't accidentally write `prod/` from a dev role. Lifecycle rules clean up dev experiments and orphaned multipart uploads automatically.

**Tech Stack:** AWS S3, IAM (customer-managed policies + principal tags), AWS CLI v2 (`aws --profile qpress --region us-east-2` always), JSON config files versioned in `infra/s3/`, bash audit script for read-only drift detection.

## Locked Decisions (W5 D-block, 2026-05-22)

- **D1 layout.** Single bucket `qpress-uploads` in `us-east-2`. Two top-level prefixes: `dev/` and `prod/`. Per-environment writes are enforced via bucket policy + principal tags (a role tagged `Env=dev` cannot write under `prod/*` even if its IAM policy were over-broad).
- **D2 SHA256.** Computed in the browser (SubtleCrypto) before the PUT. S3 validates via `x-amz-checksum-sha256` header — CORS exposes this header on response, allow-lists it on request.
- **D5 auth.** Backend uses an EC2 instance role on prod (`qpress-api-prod`), an IAM user (`qpress-dev-local`) on the owner's Mac during development. Browser has **zero** AWS credentials — it gets time-limited presigned URLs only.
- **D6.** N/A (UI-side concern — covered in W5-C).
- **Bucket name fallback.** S3 bucket names are globally unique. Step 1.1 dry-run probes whether `qpress-uploads` is taken. If so, fallback to `qpress-uploads-931886963315`. The fallback name appears in JSON files and runbook only after user confirms in Step 1.

## Task 3 — Lifecycle rules (orphan cleanup, dev expiry)

**Files:**
- Create: `infra/s3/lifecycle.json`

**Why:** Three orthogonal concerns:
1. Dev experiments shouldn't accumulate forever (30-day expire on `dev/`).
2. Aborted/incomplete multipart uploads cost money invisibly (7-day abort, all prefixes).
3. Presign-issued-but-never-completed objects on `dev/uploads-pending/` (a convention W5-B establishes — backend uploads to `dev/uploads-pending/{session_id}/` first, moves to `dev/scans/...` on `complete`) should expire after 1 day.

### Step 3.1: Draft `infra/s3/lifecycle.json`

- [ ] **Create `infra/s3/lifecycle.json`:**

```json
{
  "Rules": [
    {
      "ID": "dev-expire-30d",
      "Status": "Enabled",
      "Filter": {"Prefix": "dev/"},
      "Expiration": {"Days": 30}
    },
    {
      "ID": "abort-multipart-7d",
      "Status": "Enabled",
      "Filter": {"Prefix": ""},
      "AbortIncompleteMultipartUpload": {"DaysAfterInitiation": 7}
    },
    {
      "ID": "dev-uploads-pending-1d",
      "Status": "Enabled",
      "Filter": {"Prefix": "dev/uploads-pending/"},
      "Expiration": {"Days": 1}
    }
  ]
}
```

> **Convention pinned for W5-B.** The backend MUST PUT to `s3://qpress-uploads/{env}/uploads-pending/{upload_session_id}/{filename}` and move to `s3://qpress-uploads/{env}/scans/{scan_id}/{sha256}.{ext}` on `POST /upload/.../complete`. The 1-day rule on `dev/uploads-pending/` only kicks in if `complete` never fires. Document this in W5-B plan and `docs/db-ops.md` §7.

### Step 3.2: Dry-run — confirm pre-state

- [ ] **Run:**

```bash

  --bucket qpress-uploads 2>&1
```

Expected (fresh bucket): `NoSuchLifecycleConfiguration`.

### Step 3.3: 🚨 USER APPROVAL REQUIRED 🚨 — Apply lifecycle

- [ ] **PM runs after user "go":**

```bash

  --bucket qpress-uploads \
  --lifecycle-configuration file://infra/s3/lifecycle.json
```

### Step 3.4: Verify lifecycle post-state

- [ ] **Run:**

```bash
aws --profile qpress --region us-east-2 s3api get-bucket-lifecycle-configuration \
  --bucket qpress-uploads
```

Expected: 3 rules, all `Status: Enabled`, IDs match `dev-expire-30d`, `abort-multipart-7d`, `dev-uploads-pending-1d`.

### Step 3.5: Rollback

```bash

```

## Task 5 — Bucket policy (deny cross-prefix writes)

**Files:**
- Create: `infra/s3/bucket-policy.json`

**Why:** Defense in depth. Even if a future IAM policy mistake granted `qpress-api-prod` access to `dev/*`, the bucket policy denies any `s3:PutObject` where the principal's `Env` tag mismatches the prefix. This makes the dev/prod separation enforceable at the bucket layer regardless of IAM drift.

> **Ordering:** This MUST be applied **after** Task 4 (principals exist + are tagged). Otherwise, the deny is evaluated against principals with no `aws:PrincipalTag/Env`, which the `StringNotEquals` condition treats as "no match" — meaning the deny does not fire and the policy provides no protection during the gap. Apply order: Task 4 first, then Task 5.

### Step 5.1: Draft `infra/s3/bucket-policy.json`

- [ ] **Create `infra/s3/bucket-policy.json`:**

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyDevPrincipalsWritingProd",
      "Effect": "Deny",
      "Principal": "*",
      "Action": ["s3:PutObject", "s3:PutObjectTagging"],
      "Resource": "arn:aws:s3:::qpress-uploads/prod/*",
      "Condition": {
        "StringEquals": {"aws:PrincipalTag/Env": "dev"}
      }
    },
    {
      "Sid": "DenyProdPrincipalsWritingDev",
      "Effect": "Deny",
      "Principal": "*",
      "Action": ["s3:PutObject", "s3:PutObjectTagging"],
      "Resource": "arn:aws:s3:::qpress-uploads/dev/*",
      "Condition": {
        "StringEquals": {"aws:PrincipalTag/Env": "prod"}
      }
    },
    {
      "Sid": "DenyUntaggedPrincipalsWritingAnyPrefix",
      "Effect": "Deny",
      "Principal": "*",
      "Action": ["s3:PutObject", "s3:PutObjectTagging"],
      "Resource": [
        "arn:aws:s3:::qpress-uploads/dev/*",
        "arn:aws:s3:::qpress-uploads/prod/*"
      ],
      "Condition": {
        "Null": {"aws:PrincipalTag/Env": "true"}
      }
    }
  ]
}
```

```bash
#!/usr/bin/env bash
# Read-only audit of qpress-uploads bucket. Diffs live AWS state against infra/s3/*.json.
# Exits 0 if all PASS, 1 on first FAIL.

set -uo pipefail

PROFILE="qpress"
REGION="us-east-2"
BUCKET="qpress-uploads"
INFRA_DIR="$(cd "$(dirname "$0")/../../infra/s3" && pwd)"

pass() { printf "  \033[32mPASS\033[0m %s\n" "$1"; }
fail() { printf "  \033[31mFAIL\033[0m %s\n  %s\n" "$1" "$2"; FAIL_COUNT=$((FAIL_COUNT+1)); }

FAIL_COUNT=0

echo "=== qpress-uploads audit ($(date -u +%Y-%m-%dT%H:%M:%SZ)) ==="

# 1. Bucket exists in expected region
LOC=$(aws --profile "$PROFILE" --region "$REGION" s3api get-bucket-location \
  --bucket "$BUCKET" --query LocationConstraint --output text 2>&1) || true
if [[ "$LOC" == "$REGION" ]]; then pass "bucket region = $REGION"
else fail "bucket region" "got '$LOC' (expected '$REGION')"; fi

# 2. Public access block: all 4 = true
PAB=$(aws --profile "$PROFILE" --region "$REGION" s3api get-public-access-block \
  --bucket "$BUCKET" --query 'PublicAccessBlockConfiguration' --output json 2>&1) || true
if echo "$PAB" | grep -q '"BlockPublicAcls": true' \
  && echo "$PAB" | grep -q '"IgnorePublicAcls": true' \
  && echo "$PAB" | grep -q '"BlockPublicPolicy": true' \
  && echo "$PAB" | grep -q '"RestrictPublicBuckets": true'; then
  pass "public access fully blocked (4/4)"
else fail "public access block" "$PAB"; fi

# 3. Encryption = AES256
ENC=$(aws --profile "$PROFILE" --region "$REGION" s3api get-bucket-encryption \
  --bucket "$BUCKET" --query 'ServerSideEncryptionConfiguration.Rules[0].ApplyServerSideEncryptionByDefault.SSEAlgorithm' \
  --output text 2>&1) || true
if [[ "$ENC" == "AES256" ]]; then pass "encryption = AES256 (SSE-S3)"
else fail "encryption" "got '$ENC'"; fi

# 4. Ownership = BucketOwnerEnforced
OWN=$(aws --profile "$PROFILE" --region "$REGION" s3api get-bucket-ownership-controls \
  --bucket "$BUCKET" --query 'OwnershipControls.Rules[0].ObjectOwnership' --output text 2>&1) || true
if [[ "$OWN" == "BucketOwnerEnforced" ]]; then pass "ownership = BucketOwnerEnforced"
else fail "ownership" "got '$OWN'"; fi

# 5. CORS rule count + checksum-sha256 header allow-listed
CORS=$(aws --profile "$PROFILE" --region "$REGION" s3api get-bucket-cors --bucket "$BUCKET" --output json 2>&1) || true
if echo "$CORS" | grep -q "x-amz-checksum-sha256"; then pass "CORS allows x-amz-checksum-sha256"
else fail "CORS x-amz-checksum-sha256" "header not in AllowedHeaders / ExposeHeaders"; fi

# 6. Lifecycle: 3 rule IDs present
LC=$(aws --profile "$PROFILE" --region "$REGION" s3api get-bucket-lifecycle-configuration \
  --bucket "$BUCKET" --query 'Rules[].ID' --output json 2>&1) || true
for ID in "dev-expire-30d" "abort-multipart-7d" "dev-uploads-pending-1d"; do
  if echo "$LC" | grep -q "\"$ID\""; then pass "lifecycle rule '$ID'"
  else fail "lifecycle rule '$ID'" "not found in $LC"; fi
done

# 7. Bucket policy contains both deny statements
BP=$(aws --profile "$PROFILE" --region "$REGION" s3api get-bucket-policy \
  --bucket "$BUCKET" --query Policy --output text 2>&1) || true
for SID in "DenyDevPrincipalsWritingProd" "DenyProdPrincipalsWritingDev" "DenyUntaggedPrincipalsWritingAnyPrefix"; do
  if echo "$BP" | grep -q "$SID"; then pass "bucket policy Sid '$SID'"
  else fail "bucket policy Sid '$SID'" "not found"; fi
done

# 8. IAM policies exist
for POL in "qpress-api-s3-uploads-dev" "qpress-api-s3-uploads-prod"; do
  ARN=$(aws --profile "$PROFILE" iam list-policies --scope Local \
    --query "Policies[?PolicyName=='$POL'].Arn" --output text)
  if [[ -n "$ARN" ]]; then pass "IAM policy '$POL'"
  else fail "IAM policy '$POL'" "not found"; fi
done

# 9. Dev user exists + Env tag = dev
TAG=$(aws --profile "$PROFILE" iam list-user-tags --user-name qpress-dev-local \
  --query "Tags[?Key=='Env'].Value" --output text 2>&1) || true
if [[ "$TAG" == "dev" ]]; then pass "qpress-dev-local tagged Env=dev"
else fail "qpress-dev-local Env tag" "got '$TAG'"; fi

echo ""
if [[ $FAIL_COUNT -eq 0 ]]; then
  echo "=== ALL PASS ==="
  exit 0
else
  echo "=== $FAIL_COUNT FAIL(s) ==="
  exit 1
fi
```
