**Goal:** Close two backend punch-list items from the post-cutover sweep: (W2.1) introduce a `ProjectContext` FastAPI dependency that centralises "which project does this request operate on" and fix `POST /api/v1/projects` so an empty body creates the default `local` project; (W2.2) add a 15-second heartbeat ping to every SSE stream so an idle pipeline does not get killed by the load balancer at ~30s.

**Architecture:**
- **W2.1**: Add a `ProjectContext` dataclass + `get_project_context()` async dependency to `src/flake_analysis/api/deps.py`. The dependency resolves the project id from a path parameter named `project_id` when present, otherwise falls back to the literal `"local"`. Refit `routes/projects.py::create_project` so `CreateProjectRequest` accepts an empty JSON object (all fields optional) and the route synthesises a default `analysis_folder` from `SAA_ANALYSIS_FOLDER` when none is supplied. Keep the existing module-global `_active_project` mechanism — `ProjectContext` is the read interface; `POST /projects` is the only writer.
- **W2.2**: Extend `src/flake_analysis/api/sse.py` with an `sse_stream()` async helper that takes a `ProgressBridge` and yields encoded SSE frames, interleaving a `: heartbeat\n\n` SSE comment line every `heartbeat_seconds` (default 15) when no real event has arrived. Refit `routes/run.py`, `routes/clustering.py`, and `routes/selector.py` to call `sse_stream(bridge)` instead of inlining the `async for event in bridge.stream(): yield emit_sse_event(...)` loop. Heartbeats are SSE comments (lines starting with `:`) so existing client parsers ignore them automatically.

- httpx 0.28.1 (`AsyncClient` + `ASGITransport` for in-process streaming tests)

## Spec ambiguity resolved

These six decisions govern every task below. No later task may walk one back.

1. **`ProjectContext` is a read-only resolver, not a writer.** The dependency NEVER mutates `_active_project`. Only `POST /projects` writes. → enforced by Tasks W2.1.2 and W2.1.3.
2. **`POST /projects` empty body == `POST /projects {}` == default project.** When `analysis_folder` is omitted, the route falls back to `SAA_ANALYSIS_FOLDER` env var (then to `/mnt/analysis` literal — the same default the existing `_resolve_project_id` already uses). → enforced by Task W2.1.5.
3. **`project_id` is always `"local"` in v1.** Multi-project routing is backlog. The dependency reads `project_id` from the path param if FastAPI injected one, else returns `"local"`. → enforced by Task W2.1.2.
4. **Heartbeat interval is 15 s, hard-coded constant `SSE_HEARTBEAT_SECONDS = 15`.** Settings-level tunability is YAGNI for this punch-list item. (Domain rule: ops set the floor at 15 s; lowering requires PM signoff.) → enforced by Task W2.2.2.
5. **Heartbeat frame is a literal SSE comment** — `: heartbeat\n\n`. SSE comments are emitted as-is to the wire and ignored by EventSource clients per the WHATWG spec. → enforced by Task W2.2.2.
6. **Routes call `sse_stream(bridge)` instead of the inline drain loop.** `sse_stream` is the single seam where the heartbeat is injected. The lock+drain pattern (acquire `lock_cm` outside, release in `finally`) stays in the route — only the inner `async for` loop is replaced. → enforced by Tasks W2.2.5–W2.2.7.

"""FastAPI dependencies per backend design §1."""
from __future__ import annotations
import os
from dataclasses import dataclass
from fastapi import Request
from flake_analysis.state.manifest import Manifest, load_manifest

_active_project: str | None = None

DEFAULT_PROJECT_ID = "local"
DEFAULT_ANALYSIS_FOLDER = "/mnt/analysis"


@dataclass(frozen=True)
class ProjectContext:
    """Resolved project identity for a request.

    project_id is the opaque project handle (always "local" in v1).
    analysis_folder is the on-disk path that hosts the project's artifacts.
    """
    project_id: str
    analysis_folder: str


def _resolve_project_id(project_id: str) -> str:
    """Resolve project_id to analysis_folder path. v1: always returns _active_project."""
    global _active_project
    if _active_project is None:
        _active_project = os.environ.get("SAA_ANALYSIS_FOLDER", DEFAULT_ANALYSIS_FOLDER)
    return _active_project


async def get_project_context(request: Request) -> ProjectContext:
    """Resolve the active ProjectContext for this request.

    project_id is read from the path parameter named ``project_id`` when the
    route declares one (e.g. ``/projects/{project_id}/...``); otherwise it
    falls back to ``DEFAULT_PROJECT_ID``. analysis_folder is resolved through
    the same _active_project / SAA_ANALYSIS_FOLDER chain used by get_manifest.
    """
    pid = request.path_params.get("project_id", DEFAULT_PROJECT_ID)
    folder = _resolve_project_id(pid)
    return ProjectContext(project_id=pid, analysis_folder=folder)


async def get_manifest(project_id: str) -> Manifest:
    """Load manifest for project_id (v1: 'local')."""
    analysis_folder = _resolve_project_id(project_id)
    return load_manifest(analysis_folder)
```

"""Project lifecycle schemas per backend design §1.1."""
from __future__ import annotations
from pydantic import BaseModel

class CreateProjectRequest(BaseModel):
    """POST /projects body. All fields optional — empty body creates the default 'local' project.

    When analysis_folder is omitted the route falls back to SAA_ANALYSIS_FOLDER
    (then to /mnt/analysis literal). raw_images_dir and annotations_path remain
    optional to match the existing manifest contract.
    """
    analysis_folder: str | None = None
    raw_images_dir: str | None = None
    annotations_path: str | None = None

class ProjectHandle(BaseModel):
    """Opaque project identifier + paths."""
    project_id: str
    analysis_folder: str
    raw_images_dir: str | None = None
    annotations_path: str | None = None

class ValidatePathsRequest(BaseModel):
    """POST /projects/validate-paths body."""
    analysis_folder: str | None = None
    raw_images_dir: str | None = None
    annotations_path: str | None = None

class PathStatus(BaseModel):
    """Per-path validation result."""
    exists: bool
    is_dir: bool
    is_file: bool
    readable: bool
    writable: bool
    canonical: str

class ValidatePathsResponse(BaseModel):
    """POST /projects/validate-paths response."""
    analysis_folder: PathStatus | None = None
    raw_images_dir: PathStatus | None = None
    annotations_path: PathStatus | None = None

SSE_HEARTBEAT_SECONDS: float = 15.0
"""Heartbeat interval. Floor is 15s — lowering requires ops signoff."""

SSE_HEARTBEAT_FRAME: str = ": heartbeat\n\n"
"""SSE comment line. EventSource clients ignore lines starting with ':' per WHATWG."""


async def sse_stream(
    bridge: "ProgressBridge",
    heartbeat_seconds: float | None = None,
) -> AsyncGenerator[str, None]:
    """Drain a ProgressBridge into SSE wire frames, injecting heartbeats while idle.

    On every ``heartbeat_seconds`` interval where no real event arrives, emit
    ``: heartbeat\\n\\n`` so intermediaries (nginx / ELB) keep the connection
    alive. Real events are encoded with :func:`emit_sse_event` exactly as the
    inline drain loops did before this helper existed.

    The interval is resolved at call time via the module-level
    ``SSE_HEARTBEAT_SECONDS`` constant (so tests can ``monkeypatch.setattr``
    it). Pass an explicit ``heartbeat_seconds`` to override per-call.
    """
    interval = heartbeat_seconds if heartbeat_seconds is not None else SSE_HEARTBEAT_SECONDS
    stream_iter = bridge.stream().__aiter__()
    while True:
        try:
            event = await asyncio.wait_for(stream_iter.__anext__(), timeout=interval)
        except asyncio.TimeoutError:
            yield SSE_HEARTBEAT_FRAME
            continue
        except StopAsyncIteration:
            return
        yield emit_sse_event(event["type"], event)
```

**Note**: the call-time read of `SSE_HEARTBEAT_SECONDS` (rather than a default-argument capture) is what lets the integration test in W2.2.8 use `monkeypatch.setattr` to shorten the interval.
