## Pinned decisions

These pin the design choices the user asked the algo-engineer to make. No later task may walk one back.

1. **`ProgressCallback` canonical home: `flake_analysis.core._compat`** — leave it in `_compat.py:32` rather than relocating to a new `flake_analysis.core.types` module. Reason: every `core/pipeline/*` file already imports it from `_compat` alongside the `msg` shim (single import line per file, 6 sites), so the existing layout is already the canonical surface. Moving it elsewhere would touch 6 core files plus 6 wrapper files for zero behavior gain. The dormant-stub deletion (Task 1) shrinks `_compat.py` from 100 LOC to ~52 LOC, so the `ProgressCallback` + `msg` co-location remains tight.
2. **Thumbnail private-helper test cleanup: refactor the test, do NOT expose a public API** — split `tests/test_pipeline_thumbnails.py`. Move the 3 tests that touch `_LOCAL_CACHE_ENV`, `_should_redirect_to_local_cache`, `_local_cache_dir_for`, `_LOCAL_CACHE_ROOT` to `tests/core/test_thumbnails_internals.py`; leave the 3 wrapper-smoke tests (`test_lod_pyramid_constants`, `test_run_thumbnails_step_writes_files_and_manifest`, `test_run_thumbnails_step_caches_on_second_run`) in `tests/test_pipeline_thumbnails.py`. Reason: the local-disk cache redirect is an implementation detail of `core/pipeline/thumbnails.py` (network-mount workaround per v0.2.16 docstring `core/pipeline/thumbnails.py:21–38`); promoting `_LOCAL_CACHE_ENV` etc. to a public re-export pins it as part of the wrapper contract, freezes a workaround into the API surface, and forces api-developer / frontend-architect to think about it. `tests/core/*` is the audit-blessed home for tests that reach into `core/*` internals (§3.3 row "tests/core/test_*.py (10 files)"). The `LOD_SIZES`/`MAX_LOD` test (`test_lod_pyramid_constants`) — which uses only public module constants — also moves to `tests/core/test_thumbnails_internals.py` because those constants live in `core/pipeline/thumbnails.py` not in the wrapper, and grouping the 4 internals tests in one file is cleaner than splitting them.
3. **Parity gate after every refactor step.** Run `pytest tests/parity/test_golden_regression.py -v` after each commit. Expect 8 PASS. If any of the 8 fail, STOP — the refactor changed behavior and must be reverted.
4. **Full `pytest -q` baseline before deletions.** Before Task 1 (delete dormant stubs), capture the baseline pass count via `pytest -q 2>&1 | tail -3`. After deletion, the count must equal the baseline. The audit confirms zero call sites, but a fresh `grep -rn` plus the baseline+post-delete pytest comparison is the belt-and-braces gate.
5. **No `--no-verify` commits.** Pre-commit hooks must pass. If a hook fails, fix the issue and create a NEW commit; do not amend.
6. **No behavior change.** Zero numerics edits. Zero RNG-seeded path edits. Zero algorithm parameter changes. Pure import-graph and dead-code refactor.

def test_run_thumbnails_local_cache_redirect(monkeypatch, tmp_path):
    """End-to-end: env-var opt-in routes WebPs to local cache.

    Asserts:
      * ``index.json["cache_dir"]`` is populated and points at the
        per-analysis-folder cache directory
      * Every per-entry ``outputs[lod{N}]`` resolves under
        ``cache_dir`` (not the analysis folder)
      * The analysis-folder ``00_thumbnails/lod{N}/`` subfolders are
        NOT created (only ``index.json`` lives there)
      * The ``cache_dir`` flows through to the wrapper return value
    """
    # Redirect the cache root onto tmp so the test never touches
    # ``~/.cache/...``.
    monkeypatch.setenv(_LOCAL_CACHE_ENV, "1")
    fake_root = tmp_path / "fake_cache_root"
    monkeypatch.setattr(
        "flake_analysis.core.pipeline.thumbnails._LOCAL_CACHE_ROOT",
        fake_root,
    )

    raw_dir = _create_fixture(tmp_path, n=3)
    analysis = tmp_path / "analysis"
    analysis.mkdir()

    result = run_thumbnails_step(
        analysis_folder=str(analysis),
        raw_images_dir=str(raw_dir),
    )

    cache_dir = result.get("cache_dir")
    assert cache_dir is not None, (
        "cache_dir should be populated when redirect fires"
    )
    cache_path = Path(cache_dir)
    assert cache_path.exists(), "cache directory should be created"
    assert str(cache_path).startswith(str(fake_root)), (
        f"cache should live under monkeypatched root, got {cache_path}"
    )
    # Per-analysis-folder hashed subdir lives under the patched root
    # with the analysis-folder hash as the leaf.
    expected_cache = _local_cache_dir_for(analysis / "00_thumbnails")
    assert cache_path.resolve() == expected_cache.resolve()

    # Index file is in the analysis folder + carries cache_dir.
    out_root = analysis / "00_thumbnails"
    index_path = out_root / "index.json"
    assert index_path.exists()
    idx = json.loads(index_path.read_text(encoding="utf-8"))
    assert idx.get("cache_dir") == str(cache_path)

    # Every per-entry path resolves under the cache, not the
    # analysis folder.
    for entry in idx["entries"]:
        for lod_key, rel in entry["outputs"].items():
            resolved = cache_path / rel
            assert resolved.exists(), (
                f"thumbnail {lod_key} for {entry['raw_name']} "
                f"missing at {resolved}"
            )

    # Analysis-folder lod{N}/ subfolders should NOT have been
    # created — that's the whole point of the redirect.
    for lod in (0, 1, 2):
        assert not (out_root / f"lod{lod}").exists(), (
            f"analysis-folder lod{lod}/ should be empty when "
            "local cache is active"
        )

    # Cache subfolders should exist + carry the WebPs.
    for lod in (0, 1, 2):
        d = cache_path / f"lod{lod}"
        assert d.is_dir()
        assert len(list(d.glob("*.webp"))) == 3
